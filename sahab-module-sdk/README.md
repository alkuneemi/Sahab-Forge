# SAHAB Module SDK (قالب مستقل للمطورين الخارجيين)

هذا مستودع **منفصل تماماً** عن كود نظام SAHAB — لا يحتوي على أي من الكود
المصدري للنظام، ولا تحتاج الوصول إليه لبناء وحدة. كل ما تحتاجه موجود هنا.

## الفكرة

1. تبني وحدتك هنا وتُصدرها كملف JS واحد (`dist/module.js`) عبر `npm run build`.
2. ترفع هذا المستودع (بما فيه `dist/module.js` و`sahab.modules.json`) على
   حسابك في GitHub — **عام (public)** حتى يعمل التحميل عبر jsDelivr.
3. من داخل نظام SAHAB: `الإعدادات ← مستودع المطورين الخارجي (GitHub)`،
   أدخل اسم المالك واسم المستودع والفرع، واضغط "ربط واستيراد الوحدات".
4. تظهر وحدتك فوراً في قائمة "الوحدات" بزر تثبيت — لا حاجة لأي نشر يدوي
   أو موافقة مسبقة.

## القواعد

- **الملف `sahab.modules.json`** يجب أن يكون في جذر المستودع، ويحتوي مصفوفة
  `modules` — كل عنصر يحدّد `id` فريد و`entry` (مسار ملف الـ JS المبني).
- **الملف المبني** يجب أن يستدعي الدالة العامة
  `registerSahabModule(id, factory)` مرة واحدة (راجع `src/index.js`).
  لا تُضمّن React الخاص بك في الحزمة — استخدم `sdk.React` الممرَّر لك، وإلا
  ستتكسّر الواجهة (نسختان من React لا تتوافقان). لهذا ضبطنا
  `esbuild --format=iife` بدون تضمين React كاعتمادية.
- **لا وصول لكود النظام**: SDK يعرض لك فقط `{ React }` حالياً (سيتوسّع لاحقاً
  بحذر). وحدتك تُبنى وتُشغَّل بمعزل تام عن مصدر SAHAB.

## البناء

```bash
npm install
npm run build   # ينتج dist/module.js
```
